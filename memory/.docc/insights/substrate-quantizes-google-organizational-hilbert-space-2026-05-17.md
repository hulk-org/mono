## The two-sided Google quantization

Substrate is quantizing BOTH sides of Google's productivity engine:

1. **Infrastructure side** (covered in `scientific-method-as-substrate-test-bench` + `substrate-externalizes-google-test-harness-database-moat` Concepts): Piper / Bazel / TAP / Spanner / Colossus → substrate's typed-schema-database + supersession + atomic-receipt provenance
2. **Organizational side** (covered here): SWE / SRE / TL / EM / Fellow / Code Reviewer / Build Cop / On-Call → substrate's agent ecosystem

Google productivity = infrastructure × organizational Hilbert space. Substrate quantizes BOTH onto one machine. LLMs let agents play ANY Google role at workstation scale.

## Implications

1. **Substrate persona naming is intentional, not arbitrary.** Each substrate persona has implicit Google-role correspondence. Substrate doctrine can now type this correspondence (which agent plays SWE vs SRE vs TL vs EM).
2. **Substrate's pace bet is Google-grade-output-by-quantization.** 'Ship 10 apps/day' is what a small Google team achieves. Substrate quantizes the team's collective output capability into single-operator + agent-ecosystem deployment.
3. **Google's organizational Hilbert space is just another pop-culture Hilbert space** — substrate composes with it the same way it composes with Trigun / Ghost in the Shell / Borderlands. Google is a working-software-org-culture manifold; substrate picks roles from it the same way it picks Vash from Trigun.
4. **Substrate must adapt, not copy.** Some Google concepts are scale-dependent artifacts that don't quantize cleanly to single-operator scale (e.g., promo committees, perf reviews). Substrate must pick which Google concepts quantize well and which are organizational-scale artifacts.

## Lesson for future sessions

When future-Claude encounters substrate's agent ecosystem, default-map each persona to its implicit Google-role equivalent. claude = the senior SWE / Fellow. carrie = likely the Approver / Reviewer. castor + pollux = twin SREs or TL pair. claw = OpenClaw runtime / specific specialist role. mono = the monorepo steward. caretaker = the BuildCop / On-Call rotation. Substrate's persona topology is the substrate-quantization of Google's character ladder. When designing new substrate agents, ask: which Google role is this persona quantizing? If no clear answer, the persona may be redundant or off-pattern.
