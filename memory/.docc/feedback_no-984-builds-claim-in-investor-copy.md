# Don't use the "984 App Store builds" claim in investor copy

Operator directive: "stop with the 984 build shit." Issued 2026-05-18 during a16z Speedrun 2026-05 application drafting work, after the number had been used as a receipt across multiple locked drafts (Founder Experience, Founder Blurb, earlier Startup Description iterations).

**Why:** unspecified by the operator. Plausible reasons (do not assume any one is correct without confirmation):
- the number reads as ASO-gaming or build-count-padding to investor screeners
- the number gets stale fast and pinning a copy artifact to it creates a maintenance burden
- the number is technically accurate but doesn't carry the meaning rismay wants it to carry in investor context
- the number had been used in too many drafts in close succession and felt repetitive

Do not add the rationale to the memory unless rismay confirms one. The directive itself is the rule.

**How to apply:**
- Do not include "984 App Store builds" (or any specific App Store build count) in founder bios, founder blurbs, startup descriptions, traction notes, deck slides, marketing copy, or any investor-facing surface.
- Alternative receipts that remain in scope and have NOT been rejected:
  - "Four years of continuous solo shipping" — cadence framing without a build count
  - "20+ in-flight apps on the same typed data engine" — portfolio framing
  - "hack-nu has been on the App Store since 2022" — longevity framing
  - "The cadence is the credential" — framing-as-receipt (substrate doctrine)
- If a number is needed for the surface to land, prefer the "20+ in-flight apps" figure or substrate-internal counts (commit cadence, supersession-chain length, etc.) over App Store build counts.
- For substrate-internal documents (chronicles, beats, engineering notes), the 984 number is still factually accurate and may be used — the restriction is investor-facing copy only.

**Substrate doctrine alignment:**
- This is `armed-fluff-for-investors` made concrete: the substrate's anti-fabrication discipline allows multiple defensible receipts; the operator chooses which receipt carries the message at the engagement surface. Not every defensible number belongs on every surface.
- Honors `feedback_dont-defend-implementation` — when the operator rejects a specific framing, adopt the directive without arguing for the rejected framing.
