For wrkstrm's external-facing copy (investor forms, pitches, marketing, public-facing positioning) — lead with **what creators/users get**, not with **what we built**. Substrate vocabulary ("typed substrate," "refinement primitives," "commissioned-agent organization," "ordinality tables," "supersession-not-silent-edit") is engineering-speak that alienates the value-pattern-matching reader. Engineering substance lives as *background that justifies the value claim*, not as the foreground claim itself.

**Why:** On 2026-05-18, while drafting Speedrun-style investor copy, rismay corrected: "be less technical and more value-driven." Prior drafts had led with substrate framing (typed Swift substrate, refinement primitives, commissioned-agent organization) — that's how-we-deliver, not what-creators-get. The value-pattern reader (investor, customer, partner) reads the first phrase and stops if it's jargon. The substrate-pilled reader sees the moat anyway in the body.

**How to apply:**
- For external-facing copy, the lead must be a concrete creator/user benefit. Examples that work: "Most creators ship one app a year — we help them ship ten." "Studio-grade polish, solo-creator economics." "Your ten app ideas, all shipped."
- Examples that don't work for external copy: "typed-substrate studio," "polyglot Swift monorepo," "commissioned-agent organization with identity bundles."
- The thesis line stays as the closer, audience-translated. *"Substrate-discipline applied to creator content is the only durable moat once AI commoditizes code output"* becomes *"Creators don't need bigger teams; they need a better engine."* Same bet, value-facing language.
- This is a per-audience rule, not a global one. For *engineering peers, agent doctrine, internal substrate work*, technical vocabulary is correct and expected. The cleavage is "external/value-facing → plain English; internal/substrate work → typed vocabulary."
- Whenever rismay asks for marketing/pitch/customer-facing copy, apply this rule by default. Don't wait for the correction.
