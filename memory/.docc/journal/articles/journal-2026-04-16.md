Session focused on `wrkstrm-performance/.../graph-view-bench-press`. Ended the
session with a unified `GraphBenchBackend`-driven bench executable and a Mac
Catalyst app shell where UIKit Dynamics renders live on screen with per-probe
status. Operator ("rismay") directed the restructure; claude executed.

## What shipped

### Protocol unification

Introduced `GraphBenchBackend` in
`sources/graph-view-bench-all/GraphBenchBackend.swift`. Four static
requirements:

- `backendName: String` - short logging identifier.
- `displayName: String` - human-facing table label.
- `searchHighBound: Int` - binary-search ceiling (per-backend tuning).
- `convergenceThreshold: Int` - stop when `high - low <= this`.

Plus one isolated method:

```swift
@MainActor
func measure(
  nodes: [GraphNode],
  edges: [GraphEdge],
  durationSeconds: Double
) async -> Double
```

`@MainActor` on the requirement (not the protocol itself) so conformers that
need `nonisolated` render-thread delegate callbacks (`SCNSceneRendererDelegate`,
`MTKViewDelegate`) can coexist with the main-actor-only `measure` call. Static
`let` members on conformers are `nonisolated static let` to satisfy the
protocol's non-isolated static requirements even when the conformer class is
`@MainActor`.

Seven rendering strategies conform: `UIKitDynamicsBench`, `MetalLiveBench`,
`SceneKitLiveBench`, `SceneKitV2Bench`, `HybridBench`, `RealityKitLiveBench`,
`RealityKitSnapshotBench`.

### Consolidation: 9 executables -> 1

`Package.swift` now declares one executable product, `graph-view-bench-all`.
The 8 per-backend standalone executables were deleted; their `main.swift`
files were first renamed after their backends (`BenchPressDemo.swift`,
`MetalBench.swift`, etc.) via `@main struct` wrapping, then collapsed into
`sources/graph-view-bench-all/` as conformer classes. `BenchAll.run()`
iterates `[any GraphBenchBackend]` and runs the same 60fps binary search
against each.

Also dropped the 120fps target - binary-search per-backend runtime ~halved,
universal 60fps display-parity budget is the only target now.

### "Must render on-screen" cleanup

Per operator rule, removed conformers that do not put pixels on the display:

- `SimOnlyBench` - pure `ForceGraphSimulation.step()`, no render.
- `GpuSimBench` - `GPUForceSimulation.step()` + readback, no render.
- `MetalOffscreenBench` - Metal renders to offscreen texture.
- `SceneKitSnapshotBench` - `SCNRenderer.snapshot` offscreen.

Ported three previously-offscreen conformers to live `NSWindow` on AppKit:

- `SceneKitV2Bench` - live `SCNView` with V2 optimisations: single shared
  `SCNSphere` geometry across all nodes, per-group shared `SCNMaterial` with
  `lightingModel = .constant`, shared cylinder geometry + material for edges,
  batched per-frame transforms inside a single `SCNTransaction` with
  `disableActions = true`.
- `HybridBench` - live `SCNView`, but per-frame position updates come from
  `GPUForceSimulation.positions` (a CPU readback of a Metal compute kernel)
  rather than `ForceGraphSimulation.step()`.
- `RealityKitSnapshotBench` - live `ARView` driven by
  `ARView.scene.subscribe(to: SceneEvents.Update.self) { _ in ... }` rather than
  a manual `while / Task.yield()` loop, to compare RealityKit's native
  scheduler against our tight-loop driving pattern.

### Metal SDF circles

In `metal-game-engine/ShaderSource.swift` (authoritative) and
`Shaders.metal` (byte-for-byte mirror for editor highlighting), the quad
fragment shader now renders a signed-distance-field circle with per-pixel
anti-aliasing and no MSAA:

```
fragment float4 quadFragment(VertexOut in [[stage_in]]) {
    float d = length(in.uv) - 1.0;
    float aa = fwidth(d);
    float alpha = 1.0 - smoothstep(-aa, aa, d);
    if (alpha < 0.001) { discard_fragment(); }
    return float4(in.color.rgb, in.color.a * alpha);
}
```

`VertexOut` gained `float2 uv`; quad vertex shader writes
`cornerOffsets[vertexID] * 2.0` (mapping the corner offsets `[-0.5, 0.5]` to
`[-1, 1]` for the SDF sample). Line vertex shader initialises `out.uv =
float2(0, 0)` to keep the struct fully initialised without using SDF.

### Mac Catalyst shell

Operator repeatedly emphasised: the real baseline is UIKit Dynamics in a
Mac Catalyst shell. Implemented.

- `project.yml` now drives xcodegen (already installed at
  `/opt/homebrew/bin/xcodegen`) to declare a
  `graph-view-bench-catalyst` Mac Catalyst target pointing at the shared
  `sources/graph-view-bench-all/` directory. Settings include
  `SUPPORTS_MACCATALYST: YES`, `DERIVE_MACCATALYST_PRODUCT_BUNDLE_IDENTIFIER:
  NO`, `GENERATE_INFOPLIST_FILE: YES`, plus scene-manifest + orientation
  INFOPLIST keys.
- New UIKit shell files (all guarded `#if canImport(UIKit)`):
  `AppDelegate.swift`, `SceneDelegate.swift`, `BenchViewController.swift`,
  `UIKitBenchHost.swift`. `BenchViewController` owns a
  `benchContainer: UIView`, exposes it via `BenchHost.container`, and kicks
  off `BenchAll.run()` from `viewDidAppear`.
- AppKit-only files wrapped in
  `#if canImport(AppKit) && !canImport(UIKit)`: `CLIEntry.swift` (native
  macOS CLI `@main`), `BenchWindow.swift` (AppKit `makeBenchWindow` /
  `hideBenchWindow` / `activateBenchApp`), and six live bench files
  (`MetalLiveBench`, `SceneKitLiveBench`, `SceneKitV2Bench`, `HybridBench`,
  `RealityKitLiveBench`, `RealityKitSnapshotBench`) - operator rejected
  typealias layering and asked for "just different implementations", so each
  file is fully AppKit with no cross-platform API surface.
- `BenchAll.swift` platform-gates its backend array: Catalyst gets
  `UIKitDynamicsBench` only (pending renderer ports); native macOS CLI gets
  the six AppKit live renderers.

### Upstream fixes landed

- `PerformanceLabKit/sources/PerformanceLabKit/BenchmarkRunner.swift`: whole
  file wrapped in `#if !targetEnvironment(macCatalyst)`. `Process.run()` and
  `executableURL` are sandbox-banned on Catalyst; downstream Catalyst targets
  still get `ThresholdSearch*`, `SvgChartRenderer`, report types.
- `force-directed-graph/sources/ForceDirectedGraphUIKitDynamics/UIKitDynamicsGraphBackend.swift`:
  (1) `addLinearVelocity(_:for:)` was being passed a `CGVector`; API expects
  `CGPoint`. (2) Added `benchmark(nodes:edges:config:durationSeconds:hostView:)`
  overload; when `hostView` is non-nil, the animator's reference container
  is added as a subview of `hostView`, making the simulation visible. The
  4-arg protocol requirement stays intact and delegates to the 5-arg path
  with `hostView: nil`, so standalone callers are unaffected.

### Status hook

`BenchStatusDisplay` in `UIKitBenchHost.swift` is a trivial closure-sink
pattern:

```swift
@MainActor
enum BenchStatusDisplay {
  static var sink: ((String) -> Void)?
  static func announce(_ text: String) { sink?(text) }
}
```

`BenchViewController.viewDidLoad` assigns a sink that pipes text into a
`statusLabel`. `UIKitDynamicsBench.measure` calls `announce(...)` per probe -
the other six live renderers don't announce yet (on macOS the label isn't
visible anyway; they'll announce when their UIKit siblings land).

## Errors + fixes worth remembering

- `_NSWindowTransformAnimation` dealloc SIGSEGV during rapid binary-search
  probe transitions. Root cause: `NSWindow.close()` triggered
  `_NSWindowTransformAnimation` instances that the CoreAnimation transaction
  commit later tried to release against stale pointers. Fixed by setting
  `win.animationBehavior = .none` at creation and using `win.orderOut(nil)`
  instead of `close()` in `hideBenchWindow`.
- SceneKit live window rendered empty. Root cause: `SCNCamera` default
  `zFar = 100`, but the camera was placed at `z = 500` looking at a `z = 0`
  graph plane, so every node was clipped by the far plane. Fixed with
  explicit `camera.zNear = 1; camera.zFar = 1000`.
- `@main struct BenchAll.main() async` caused isolation errors when nested
  `logProbe` was inferred `@MainActor @Sendable` and failed to pass as a
  `nonisolated` closure to `findSixtyFpsBreakpointAsync`'s `onProbe`
  parameter. Fixed by hoisting `log` + `logProbe` to file scope (nonisolated),
  and marking `findSixtyFpsBreakpointAsync` `@MainActor` to match the
  main-actor live backends it captures.
- Catalyst window was empty even though UIKit Dynamics physics was running.
  Root cause: library created its animator container as an orphan UIView
  ("positioned offscreen container, never displayed" per its own comment).
  Fixed upstream via `hostView:` parameter.

## Measurements captured

Native macOS CLI run, 60fps binary search, 2s per probe:

| Backend                       | 60fps  | Notes                               |
|-------------------------------|-------:|-------------------------------------|
| Metal (live window)           | ~9375  | Hit `searchHighBound = 10_000`      |
| Hybrid GPU+SCN (live)         | ~1971  | GPU sim + live SCN - ~6x pure SCN   |
| SceneKit (live window)        |  ~323  | -                                   |
| SceneKit V2 (live window)     |  ~322  | V2 opts yielded ~0 improvement      |
| RealityKit (live window)      |  ~323  | -                                   |
| RealityKit (scheduler)        |  ~318  | Scheduler ~= manual loop             |
| UIKit Dynamics (Catalyst)     | pending | Catalyst measurement run TBD        |

Three findings worth keeping:

1. SceneKit V2 optimisations moved the needle by one node. Shared geometry
   + constant lighting + batched transactions don't help because SceneKit's
   real cap at ~320 nodes is deeper - scene-graph traversal or render
   encoding, not per-node setup.
2. Hybrid (GPU sim + live SCN, ~1971) x plain SceneKit (CPU sim + live SCN,
   ~323) isolates the sim cost: CPU O(n2) sim was eating most of the frame
   budget at ~323 nodes. SceneKit's render alone tops out around ~2000.
3. RealityKit's native `SceneEvents.Update` subscriber and a manual
   `Task.yield()` loop produce identical throughput. Pick whichever reads
   cleaner.

## Future plans

1. **Port live renderers to UIKit** - Metal, SceneKit, SceneKitV2, Hybrid,
   RealityKit, RealityKitSnapshot each get a `#if canImport(UIKit)` sibling
   using `BenchHost.container` + `BenchStatusDisplay.announce`. Surface
   changes are `NSColor -> UIColor` and window hosting only; MTKView, SCNView,
   ARView exist on both frameworks with the same type name.
2. **Full Catalyst comparison table** - once renderers are UIKit-native,
   Catalyst run produces row-0 = UIKit Dynamics baseline with every other
   backend compared against it in a "vs UIKit Dynamics" column.
3. **Raise Metal `searchHighBound`** - current 10 000 convergence at the
   ceiling; true 60fps breakpoint is above. Bump to 50 000 or 100 000 to
   find the real number.
4. **Per-backend `BenchStatusDisplay.announce(_:)`** - every conformer
   should announce per probe for UI parity, not just `UIKitDynamicsBench`.
5. **Metal visual quality parity** - SDF anti-aliasing is in place; still
   want to compare MSAA 4x (SceneKit's approach) vs SDF on the same node
   counts.
