# 2026-05-08 — Hello World and the full stack

## The thesis

The operator said: build a Hello World. Google style. Ship it to the world.

We started with eleven lines of Swift. Two Text views. A VStack. That's it.

Then we started asking what "ship to the world" actually means.

Thirty-nine locales. Right-to-left Arabic and Hebrew. Dynamic Type at twelve
sizes. WCAG contrast on every color. An app icon. A real bundle ID. Privacy
manifest. App Sandbox. Code signing. Firebase. Remote config. Kill switches.
Feature flags. A design token system in three output formats. A QA receipt
with twenty-four screenshots. A launch calendar tracking thirty-three gates
for a Type 2 launch.

All for two words.

## What we built

**service-universal** — a new collective. Seven protocol packages, each a
clean abstraction with a no-op bundled fallback:
- `common-remote-config` (Firebase behind a protocol)
- `common-analytics`, `common-feature-flags`, `common-crash-reporting`
- `common-font` (WrkstrmFont-backed, role-based token system)
- `common-network` (NWPathMonitor), `common-diagnostics` (Inspectable)

`wrkstrm-font` and `wrkstrm-color` migrated from `wrkstrm-components` into
`service-universal`. They were primitives masquerading as components.

**design-token-schemas** — W3C DTCG token model with a Swift CLI that reads
`google.tokens.json` and emits Swift enums, CSS custom properties, and Android
XML. One source, every platform. The WCAG audit subcommand runs automatically:

```
✗ color.yellow  #FBBC05  on white: 1.71:1 FAIL ← known gate blocker
✓ color.green   #34A853  on black: 6.87:1 AA
```

**launch-calendar** (renamed from release-calendar) — the full gate model.
Thirty-five LaunchGate cases. Each case carries `requiredArtifacts`: the exact
file or document that proves the gate is done. `LaunchGateEvaluator` produces
a per-gate `GateDecision` (Go / No-Go / Conditional / Missing) and an overall
`LaunchDecision`. The evaluator is a pure function on the schema — same code
in the app and in the `launch-gate-owl` CLI Koma.

**QA harness** — three layers:
1. Mathematical contrast tests (`withKnownIssue` for yellow)
2. XCUITest pseudo-localization + Dynamic Type permutation suite
3. `QAReportGenerator`: 24-permutation snapshot matrix → JSON receipt
   Markdown report + 24 PNGs committed to `qa-receipts/`

The receipt opens as a Markdown document with every screenshot inline. The
`testPlanReady` gate flips to `passed` when the receipt is signed.

## The pedagogy insight

Halfway through, the operator named it: this is pedagogy.

The `LaunchArtifactGuard` crashes your DEV build with:

```
DEV guard: file launch.docc/index.md before running.
```

You don't read documentation about what a launch doc is. You hit the wall.
The wall tells you exactly what to do. You do it. The wall goes away. You
learned something.

Every gate is a lesson in the same form. The `wcagContrast` gate doesn't
just say "accessibility". The WCAG test says "1.71:1 — fails AA large text."
The token file says "FBBC05 on white — needs ≥3.0:1." The design-token-cli
audit says exactly which token fails and what ratio it has.

The system teaches by blocking, not by telling.

## The factory

"We are a Digikoma factory after all."

Every gate that can be automated should have a Koma. The Koma reads logs from
the running system via adapters. It produces a structured receipt. The
`gate-sync-ant` reads all receipts and updates the launch calendar. No gate
requires manual JSON editing.

CommonLog is already the logging backbone — wired into MetricKitCrashReportingService
today. The adapters fan structured logs to the right specialist:
- `ContrastAuditAdapter` → `contrast-audit-owl`
- `CrashLogAdapter` → `crash-analysis-owl`
- `AnalyticsAdapter` → `instrumentation-ant`

Foundation Models on-device synthesizes failures into fix suggestions.
The developer writes code and watches the gates close.

"You build. Everything else is taken care of."

"This is going to free you."

on the creative, high-judgment work. The infrastructure handles the ceremony.
