# 2026-04-20 — Swarm investigation, Beads discovery, Workflow app

## Session arc

Started with brew update guidance and a broken Codex skills symlink.
Pivoted to a deep investigation of christopherkarani/Swarm (Swift agent
framework) vs Digikoma. The investigation surfaced the key architectural
difference: Swarm fuses navigation + execution (LLM controls both),
Digikoma separates them (spec controls navigation, FM controls execution
at each node). The user corrected my initial framing that Digikoma "doesn't
use LLM" — it does, at every node, via FoundationModels.

Built digikoma-web-fetch and digikoma-summarize-context to run an equivalent
research pipeline experiment. Ran three iterations (v1 naive → v2
chrome-filtered → v3 clean HTML extraction). Output improved from
marketing slop to actual technical content, but the chrome filter still
drops two of three test pages.

Mid-session discovery: Beads (gastownhall/beads) is the missing work
management layer for wrkstrm apps. The app will be called Workflow.
The full stack: Workflow (UI) → Beads (what + when) → Ghost (bridge)
→ Digikoma (how). Dolt is the shared operational data layer (git for code,
Dolt for work state). sqlite-data (pointfreeco) enables iOS sync.

Created beads-schemas v0.1.0 with ~40 types validated against the Go
source (100+ structs). Scaffolded the Workflow app, built and launched
it. Connected WorkstreamTemplateModel to Beads by moving paperclipIssue
to per-task and adding beadsIssueId. Renamed PaperclipIssueProtocolModel
to PaperclipIssueModel across the schema chain.
