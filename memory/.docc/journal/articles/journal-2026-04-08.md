## study-lab build chain

The build chain that finally reached `** BUILD SUCCEEDED **` for
`study-lab.mac` required peeling 10 layers:

1. interview-prep → study-lab rename (65 files)
2. clia-chat path fix (clia-org → clia-app-org after the earlier move)
3. clia-chat-swiftui cross-collective `clia-ollama-common-ai` path fix
4. wrkstrm-kit relocated to wrkstrm-components home and tagged v3.0.0
5. 8 wrkstrm-kit consumers rewired
6. gen-ui-core common-process via `localOrRemote` (was hardcoded local path)
7. docc-browser-macos-kit `@_exported import DocCBrowserKit` so the demo
   can see DocCBrowserStore via the macos kit alone
8. 7 SPM upstream release tags pushed
9. Stale `Package.resolved` files in `wrkstrm` deleted
10. **`SPM_USE_LOCAL_DEPS=true` shell env var leak unset.** This was the
    sneakiest one — it was set in the user's shell from earlier in the
    session and made every SPM manifest evaluation pick the local arm of
    `localOrRemote`, which then resolved relative paths from the
    `~/Library/Developer/...` checkout dirs and produced confusing
    "stable depends on unstable" conflicts. Solved by `env -u
    SPM_USE_LOCAL_DEPS xcodebuild ...`.
11. Reverted 12 over-renamed schema-exported `InterviewPrep*` types in
    `study-lab` sources.

Final command that worked:

```sh
cd codeswiftly/private/universal/apple/apps/study-lab
env -u SPM_USE_LOCAL_DEPS xcodebuild -project study-lab.xcodeproj \
  -scheme study-lab.mac -destination 'platform=macOS' \
  -configuration Debug build
# ** BUILD SUCCEEDED **
```

## Lessons saved to memory

This session generated multiple feedback memories at
`~/.claude/memory/.docc/`:

- `feedback_no-recursive-grep.md` — never `bash grep -rln` across
  `collectives/`; it hangs 20-30+ minutes. Always use the Grep tool
  (ripgrep). Burned 31 minutes of operator time on this in this session.
- `feedback_swift-not-python.md` — never use `python3 - <<'PY'` heredocs in
  this substrate. Swift is the only sanctioned helper language.
- `feedback_no-bash-scripts.md` (added by operator) — bash for-loops,
  while-read pipelines, multi-step shell logic are also forbidden. Save
  anything beyond a discrete command as a Swift CLI.
- `feedback_localOrRemote-default-false.md` — `useLocalDeps` must default
  `false`; only `true` on explicit truthy `SPM_USE_LOCAL_DEPS`. Never
  invert.
- `feedback_open-means-launch.md` — for Apple projects, "open it" means
  launch the built `.app`, not open the `.xcodeproj` in Xcode.
