## The arc

The session opened on rework of identity/persona work and quickly
pivoted into rethinking the substrate's planning ontology. The
operator's reveals came in roughly this order:

1. *milestones should link a thread. that feels right.* → typed
   thread-back-pointers as load-bearing closure.
2. *the PRD and CUJ are the middle. everything starts with ideation.
   an apple falling on Newton's head.* → ideation is the genesis;
   PRD/CUJ are the conversion point, not the start.
3. *the work wave is trying to be generic.* → the substrate's
   typed-shapes pull toward genericness because the territory they
   map is timeless.
4. *we have the thread which is the direction. the work vs knowledge
   graph going above or below some weird type of axis through time.*
   → dual-graph topology with the Thread as the temporal axis.
5. *the gym, school, museum, zoo are the knowledge graph surfaces.
   and the app types, OSs surfaces are the work graph surfaces. I
   would dare call them interfaces.* → physical-digital × knowledge-
   work 2×2 grid, with interfaces as the typed surfaces.
6. *we should be trying to use single variable names in the json
   schema. saves space + leaves room for proper dictionary
   projections later.* → compact wire keys as substrate doctrine.
7. *canonical swift in code. small json. fast query in sqlite.
   compact storage in fleece.* → the four-layer stack made explicit.
8. *there is also the monitor phase we handle that too.* → MONITOR
   added to the chain.
9. *that is what the presence is about.* → monitor splits into live
   presence layer + aggregate monitor layer.
10. *there has to be brain storming concepts and time boxes and
    conceptualizations.* → IDEATION interior unpacked into 5
    sub-phases.
11. *we need to distinguish between a universal computer system
    truth and truths about certain systems... like games vs real
    world vs theoretical. whoa. that is so crazy.* → substrate
    reframed as meta-system hosting multiple truth-systems.
12. *these are called hilbert spaces. logical hilbert spaces.* →
    truth-systems typed as logical Hilbert spaces (axioms = basis,
    theorems = linear combinations, projections between systems).

Each reveal compounded with the prior ones. The substrate's typed
discipline kept absorbing them coherently — every "whoa that is so
crazy" moment turned out to be the substrate's pre-existing shape
finally being recognized at its proper depth.

### PRD + CUJ v0.1.0 — in-place discipline refit

Both refined to substrate-grade discipline. PRD's JSON schema
shrank from verbose-keyed to compact-keyed wire shape (`slug→i`,
`title→t`, `problemStatement→s`, `functionalRequirements→fR`,
etc.). String timestamps replaced with `chrononID: Int64`. Free-string
persona refs replaced with typed LinkRef vault-record targets. KomaOrg
typed as integer ordinal. Inline `PRDMilestone` removed in favor of
LinkRef references to canonical milestone-schemas. Typed NFR concerns
(performance / security / accessibility / privacy / reliability /
observability / compliance / localization) with structured NFRValue
substructure. Typed NonGoal with mandatory rationale. ThreadLink for
originating thread.

`PRDModel.swift` rewritten completely: full readable Swift property
names (`slug`, `targetUsers`, `originatingThreadRef`,
`functionalRequirements`, etc.) with `CodingKeys` mapping to the
compact wire keys (`i`, `u`, `oT`, `fR`). Local `LinkRefModel` and
`ThreadLinkModel` mirrors embedded in the package for lightweight
use without external SPM deps; documented for future migration to
substrate-canonical primitives when their dependencies are
formalized. Typed enums (PRDStatus / PRDGoalRank / PRDRiskSeverity)
with Int rawValue for ordinal round-trip. The op caught me partway
through writing single-letter Swift property names and corrected
with the canonical-Swift / small-JSON doctrine — clean rewrite.

### Swift-universal — typed-substrate-system architecture-docc bundle

Standalone DocC bundle at
`swift-universal/typed-substrate-system.architecture.docc/`, modeled
on `state-machines.architecture.docc`. Tiered Swift Universal
adoption standard for typed substrate systems:

```
Tier 0 — Property Documentation (chronon-grade records/goal/loss/
        query/witness/proof ledger per property)
Tier 1 — Per-Record Fixture Coverage (positive/negative/counterfactual)
Tier 2 — Collision Rule Corpus (typed forbidden-pair records)
Tier 3 — Entailment Rule Corpus (typed if-then records)
Tier 4 — Coherence Runner Integration (substrate-wide one-pass)
```

Seven playbooks shipped (~1,900 lines total):

1. **audit-before-scaffold-discipline** — five-times-validated
   pattern in this session alone. Each new family proposal triggers
   enumeration of existing families first; four of five session
   audits concluded "do not scaffold new — existing covers it."
2. **primitive-extraction-discipline** — small types so primitives
   stay primitive. Seven primitive extractions emerged in this
   session (chronon-range, actor-ref adoption, falsification-
   condition, weighted-claim, deliverable-kind, expected-failure,
   breakage-record).
3. **typed-orthogonal-dimensions** — KomaOrg / SpecificationKind /
   PathKind / DeliverableKind / MilestoneKind compose independently;
   sub-tag escape hatch for fine-grained distinctions.
4. **collision-rule-discipline** — substrate-grade property-based
   testing via typed forbidden-pair records. 11 concrete collisions
   enumerated as starter corpus.
5. **coherence-as-frame-of-understanding** — one-pass typed
   validation as the substrate's self-check. Three layers (LDT
   repl-proofs / per-family fixture validity / substrate-wide
   cross-family coherence) integrated.
6. **four-layer-stack-discipline** — canonical Swift / small JSON /
   fast SQLite / compact Fleece projection chain. CodingKeys is the
   seam between Layer 1 and Layer 2; projections are lossless in
   the round-trip direction.
7. **typed-failure-epistemology** — failures as first-class
   substrate records. Three-layer typed-failure surface
   (CollisionRule declarative / ExpectedFailure witness /
   BreakageRecord incident). Ternary recovery state:
   open-active / accepted-as-eternal / resolved.

Cross-link added to `swift-universal/.docc/index.md` naming the new

vs lifecycle truth.

### Truth-system meta-theory (in beads, not yet scaffolded)

The session's most far-reaching reveal. The substrate isn't hosting
a single typed planning ontology — it's a meta-system that hosts
**multiple typed truth-systems**, each with its own modality and
falsification rules:

- universal-computer-truths (convention-we-commit-to)
- theoretical-truths (necessary — math, logic, type theory)
- real-world-truths (contingent — physical, social, biological)
- game-world-truths (permitted-by-rules)

Operator named these as **logical Hilbert spaces**: axioms = basis
vectors, theorems = linear combinations of axioms, orthogonality =
logical independence, projection operators between systems with
documented information loss, completeness as Gödel-bounded ideal.
The substrate's coherence runner becomes a typed self-check on
per-system consistency.
