Decided 2026-04-12. Logikoma is the class. The slug is the individual.

**Why:** "Komo" alone is too generic — it's just the chassis/body. "Logikoma" carries the logic contract that makes it composable (pipeline, tree, graph patrol). The name comes from Ghost in the Shell: ARISE's Logicoma (ロジコマ), the logistics think-tank with moderate AI. Our Logikoma are the same shape: bounded, logic-bearing, mobile, composable.

**How to apply:**
- When referring to the class of execution units: "Logikoma" (not "Komo" alone)
- When referring to the chassis/runtime: "Komo" is acceptable but secondary
- When referring to an individual: use its slug ("the echo Logikoma", "vision-scan")
- The substrate directory remains `koma/` (filesystem convention)
- The schema family remains `digikoma-schemas` (already shipped)
- The org remains `digikoma-org` (already created)
- The organism aspect remains `DigikomaAspect` in the schema (backward compat)
- But documentation, conversation, and new code should use "Logikoma" as the class noun

```
タチコマ (Tachikoma) = lore ancestor
コマ (Komo)           = chassis / body / runtime
ロジコマ (Logikoma)   = the class — chassis + logic contract
slug                  = the individual
```
