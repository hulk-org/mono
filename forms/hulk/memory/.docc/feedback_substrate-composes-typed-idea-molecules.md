**Why versioning matters now:**

- schema-universal already versions schema families (v0.1.0 → v0.2.0 with typed migration packages)
- Workstreams-as-Swift-Packages inherit this: Workstream<SpawnSoftware>@v0.1.0, v0.2.0, etc.
- Each workstream version = Swift Package release with semantic versioning
- Migration packages between workstream versions track typed evolution
- Workstreams compose via Package dependencies WITH version constraints
- A workstream improvement = Swift Package version bump with typed receipts (test outputs)

**The substrate is a PACKAGE MANAGER FOR IDEAS.** SwiftPM + Git tags + DocC + tests = full lifecycle for every typed idea. The same discipline that ships software packages now ships idea packages.
