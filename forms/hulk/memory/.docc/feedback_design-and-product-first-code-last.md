**The substrate's typed work sequence** (positions are non-negotiable):

```
1. Design + Product   ← conversational + conceptual + visual + structural + ontological + intent work
2. PRD                ← typed substrate-canonical product-requirements record
3. CUJ                ← typed critical user journey enumeration
4. CODE               ← implementation derived from positions 1-3
5. QA                 ← typed coding-QA evidence with regression discipline
6. maybe LAUNCH       ← only when every part of coding-QA passes
```

**Code is at position 4, not position 1.** Earlier session-shipped doctrine ("feedback-issue-found-means-entire-spawn-software-rerun") was incomplete — it started at PRD. This axiom adds Design + Product at position 1 where it belongs.

**What "Design + Product" includes** (this is the EXPENSIVE work — substrate's investment):

- Conversational threads where operator + agent explore the problem
- Conceptual framing (what is this thing, why does it exist)
- Visual hierarchy + navigation grammar
- Audience-packet stack (who is this for)
- Ontology-review work (concept naming)
- Brand-identity records (how it presents)
- Design-truth drafts (pre-PRD)
- Intent + outcome articulation

These are PRIMARY substrate work. Code that implements them is downstream.
