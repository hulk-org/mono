In review-corpus apps (EGM Compendium and any wrkstrm-games app surfacing EGM data), an entry's ranking is the most newsworthy piece of metadata. It belongs at the **top** of the detail page as a hero element — large rank badge, accent color, near the title — not buried below description / developer / genre / pages / era.

**Why:** rismay said "we should be proud of that 9" about Super Metroid's #9 in the 1000-best list. The ranking is what makes the entry notable; the rest (developer, genre, magazine page) is supporting metadata. The source magazine's layout already treats rank as hero (the "09" badge is built into the image asset). A wrapper UI that displays the same fact as a small text line at the bottom is fighting the source material's editorial intent.

**How to apply:** when designing or reviewing a detail page for any EGM-corpus entity (game / magazine asset / dev / era / genre), put the highest-applicable ranking immediately under the title or as a corner badge. If the entry has multiple rankings (e.g. #9 in 1000-best + #3 in SNES-only), show the top-tier ranking large, others as a stack. Don't let "Rankings" appear as a section heading three scrolls down.
