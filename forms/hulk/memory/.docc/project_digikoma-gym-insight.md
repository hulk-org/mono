The gym is the thing that makes the whole organism learnable rather than just deployable.

The moment Stage World + Seatbelt + tool allowlist composed on 2026-04-15. Key insight: the fixture IS the prompt. The gym is a harness for on-device Foundation Models inference without writing inference code yet.

**Why:** The loop — design graph → stage context → run koma → inspect receipt → iterate — closes the gap between koma spec design and real session behavior. The allowlist maps directly to Foundation Models tool registration. The Seatbelt profile is an audit-trail capability certificate.

**How to apply:** Before wiring any real Foundation session, always ask: can this be gymmed first? Stage the context the session would see, run the koma that would handle it, inspect the handoff. Only graduate to live inference when the gym passes.
