- **Path-scoped `git commit <path>` discipline** in a workspace with an auto-commit hook that sometimes sweeps unrelated working-tree state. Never `git add -a` or `git commit -a`. Always name the specific paths in the commit invocation; the path-scoped form bypasses the index and captures only the named path's current working-tree content, leaving pre-staged unrelated changes (e.g. `.gitmodules` URL renames staged by other processes) untouched. Used twice in the same session to avoid bundling a `getyourguide` submodule URL rename into two different shipping commits.
