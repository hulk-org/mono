Authored 2026-05-17 with the operator after tonight's ghost.app voice
cloning planning-ontology walk + operator-app painting walk. The
operator named the pattern and asked for it codified.

**Why:** Substrate sessions repeatedly fall into two failure modes —
premature scaffolding (agent authors typed records before the operator's
mental shape is fully named) and agent-narrated state (agent reports
work as done without typed evidence; OpenClaw incident is canonical).
The SOP makes the two-mode workflow explicit instead of vibes-based.

**How to apply:**

1. On session open, after loading standard substrate startup surfaces,
   ALSO read this SOP. It composes with `AGENTS.md`, `CLAUDE.md`, and
   commissioned home docs; it does not replace them.
2. Default to IDEATION mode (open-prose painting, substrate-sourced
   questions, held canvas, no typed records yet) when the operator opens
   a new substrate spark with open prose.
3. Default to CREATION mode (typed-models-first, audit-before-scaffold,
   surface schema-family gaps as beads, drift surfacing, digikoma-git
   tight commits) when the operator names a path explicitly.
4. When in doubt about which mode is active, ASK. Operating in the wrong
   mode is the substrate's worst-case waste.
5. Modal-switch triggers are operator-initiated only — `"b"`, `"a.b. do
   that actually"`, `"let's do it"`, `"scaffold it"`, `"wait, i was
   thinking about..."` are all canonical signals. Never switch modes
   unilaterally.
6. Cite the SOP in commit messages when authoring records that came
   through its discipline (e.g., `"Author PRD per
   substrate-coherent-ideation-and-creation.sop.docc Mode 2 discipline
   (audit before scaffold; surfaced commercialRightsFlag drift for
   operator review)"`).
