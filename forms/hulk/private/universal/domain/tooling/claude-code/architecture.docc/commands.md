# commands

The `commands/` directory contains the slash-command tree (~90
entries). The registry that composes them is `commands.ts` at
`src/` root.

## Registry shape

`commands.ts` imports each command by name from
`./commands/<name>/index.js` (or `./commands/<name>.js` for the
single-file commands). Like `tools.ts` and `tasks.ts`, it is a
**static composition file**, not a dynamic loader. The
`assist/source/organizeImports` biome rule is suppressed at the top
because the order encodes ant-only markers that the bundler reads.

## Single-file commands at root

- `advisor.ts`, `brief.ts`, `bridge-kick.ts`, `commit.ts`,
  `commit-push-pr.ts`, `createMovedToPluginCommand.ts`.

`createMovedToPluginCommand.ts` is the **deprecation shim factory**:
when a command moves into the plugin system, the old command name
becomes a stub generated by this helper.

## Cross-cutting

- **Plugin migration is a real lifecycle.** `createMovedToPluginCommand`
  exists because commands move *out* of this tree into the plugin
  system. Don't assume a command's home is permanent.
- **`good-claude`, `btw`, `bughunter`, `thinkback`, `thinkback-play`**
  are all flavor / feedback / experimental commands that hint at the
  product surface beyond the obvious tooling.
- **Two ant-trace entries.** `ant-trace/` (command) and the import-order
  marker in the registry. Ant-only debug.
